# Drink Lab API Contract

## 1. API Purpose

Drink Lab API is a REST API for managing a drink menu. It allows clients to create, read, update, delete, and sort drink records.

The API currently supports basic CRUD operations for drinks.

---

## 2. Base URLs

### Local Development

```txt
http://localhost:3000
```

### Production

```txt
https://<your-render-service-name>.onrender.com
```

---

## 3. Resource: Drink

A drink has the following fields:

| Field        | Type     | Required | Description                        |
| ------------ | -------- | -------: | ---------------------------------- |
| `id`         | integer  |       no | Generated by the database          |
| `name`       | string   |      yes | Name of the drink                  |
| `category`   | string   |      yes | Base spirit or category            |
| `alcoholic`  | boolean  |      yes | Whether the drink contains alcohol |
| `created_at` | datetime |       no | Created timestamp                  |
| `updated_at` | datetime |       no | Updated timestamp                  |

---

## 4. Supported Categories

The `category` field must match one of the accepted drink categories.

Supported values include:

```txt
vodka
gin
rum
white_rum
tequila
mezcal
whiskey
bourbon
rye
scotch
brandy
cognac
pisco
soju
sake
liqueur
amaro
aperitif
fortified_wine
wine
beer
champagne
absinthe
non_spirit
```

Category input is normalized to lowercase.

Example:

```json
{
  "category": "Tequila"
}
```

May be stored and returned as:

```json
{
  "category": "tequila"
}
```

# Endpoints

---

## 5. GET /api/v1/drinks

Returns all drinks.

### Request

```http
GET /api/v1/drinks
```

### Query Parameters

| Parameter | Required | Description                       |
| --------- | -------: | --------------------------------- |
| `sort`    |       no | Sorts drinks by a supported field |

Supported sort values:

```txt
name
category
date_added
date_edited
```

### Example Requests

```bash
curl http://localhost:3000/api/v1/drinks
```

```bash
curl http://localhost:3000/api/v1/drinks?sort=name
```

```bash
curl http://localhost:3000/api/v1/drinks?sort=category
```

```bash
curl http://localhost:3000/api/v1/drinks?sort=date_added
```

```bash
curl http://localhost:3000/api/v1/drinks?sort=date_edited
```

### Success Response

Status:

```txt
200 OK
```

Body:

```json
[
  {
    "id": 1,
    "name": "Margarita",
    "category": "tequila",
    "alcoholic": true,
    "created_at": "2026-06-10T04:57:25.806Z",
    "updated_at": "2026-06-10T04:57:25.806Z"
  }
]
```

### Empty Response

If no drinks exist, the API returns an empty array.

Status:

```txt
200 OK
```

Body:

```json
[]
```

## 7. POST /api/v1/drinks

Creates a new drink.

### Request

```http
POST /api/v1/drinks
```

### Headers

```http
Content-Type: application/json
```

### Request Body

```json
{
  "name": "Margarita",
  "category": "tequila",
  "alcoholic": true
}
```

### Example Request

```bash
curl -X POST http://localhost:3000/api/v1/drinks \
  -H "Content-Type: application/json" \
  -d '{"name":"Margarita","category":"tequila","alcoholic":true}'
```

### Success Response

Status:

```txt
201 Created
```

Body:

```json
{
  "id": 1,
  "name": "Margarita",
  "category": "tequila",
  "alcoholic": true,
  "created_at": "2026-06-10T04:57:25.806Z",
  "updated_at": "2026-06-10T04:57:25.806Z"
}
```

### Error Responses

If `name` is missing:

Status:

```txt
422 Unprocessable Content
```

Body:

```json
{
  "errors": [
    "Name can't be blank"
  ]
}
```

If `name` already exists:

Status:

```txt
422 Unprocessable Content
```

Body:

```json
{
  "errors": [
    "Name has already been taken"
  ]
}
```

If `category` is invalid:

Status:

```txt
422 Unprocessable Content
```

Body:

```json
{
  "errors": [
    "Category is not included in the list"
  ]
}
```

If `alcoholic` is missing:

Status:

```txt
422 Unprocessable Content
```

Body:

```json
{
  "errors": [
    "Alcoholic is not included in the list"
  ]
}
```

## 8. PATCH /api/v1/drinks/:id

Updates an existing drink.

### Request

```http
PATCH /api/v1/drinks/:id
```

### Headers

```http
Content-Type: application/json
```

### Request Body

The client may send one or more fields to update.

```json
{
  "name": "Margarita Picante",
  "category": "tequila",
  "alcoholic": true
}
```

### Example Request

```bash
curl -X PATCH http://localhost:3000/api/v1/drinks/1 \
  -H "Content-Type: application/json" \
  -d '{"name":"Margarita Picante"}'
```

### Success Response

Status:

```txt
200 OK
```

Body:

```json
{
  "id": 1,
  "name": "Margarita Picante",
  "category": "tequila",
  "alcoholic": true,
  "created_at": "2026-06-10T04:57:25.806Z",
  "updated_at": "2026-06-10T05:15:10.123Z"
}
```

### Error Responses

If the drink does not exist:

Status:

```txt
404 Not Found
```

Body:

```json
{
  "errors": [
    "Couldn't find Drink with 'id'=\"999\""
  ]
}
```

If the update contains invalid data:

Status:

```txt
422 Unprocessable Content
```

Body:

```json
{
  "errors": [
    "Name can't be blank"
  ]
}
```

If the updated name already exists:

Status:

```txt
422 Unprocessable Content
```

Body:

```json
{
  "errors": [
    "Name has already been taken"
  ]
}
```

# 10. Status Code Summary

|                 Status Code | Meaning                                      | Used When                                                                  |
| --------------------------: | -------------------------------------------- | -------------------------------------------------------------------------- |
|                    `200 OK` | Request succeeded                            | Reading or updating drinks                                                 |
|               `201 Created` | Resource created                             | Creating a drink                                                           |
|            `204 No Content` | Resource deleted successfully                | Deleting a drink                                                           |
|             `404 Not Found` | Resource does not exist                      | Looking up, updating, or deleting a missing drink                          |
| `422 Unprocessable Content` | Request was understood but failed validation | Missing name, duplicate name, invalid category, or missing alcoholic value |

---

# 11. Error Response Format

All validation and not-found errors return an `errors` array.

Example:

```json
{
  "errors": [
    "Name can't be blank"
  ]
}
```
